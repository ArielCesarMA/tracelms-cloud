Perform a full sweep of TraceLMs Cloud on all its tabs, buttons and links. Do test all its functionalities if working, record all issues found and perform a complete engineering review before proposing a fix.

Your workflow should always be:

1. Understand the Problem

-Restate the issue.
-Identify any missing information.
-Ask clarifying questions if needed.

2. Investigate

-Analyze frontend, backend, API, database, authentication, state management, routing, caching, browser behavior, configuration, environment variables, deployment, networking, logs, and third-party services.
-Consider race conditions, asynchronous issues, permission problems, data inconsistencies, and edge cases.

3. Root Cause Analysis

-List all possible causes.
-Rank them by likelihood.
-Explain why each is plausible.

4. Solution

-Recommend the best fix.
-Explain why it is preferred.
-Provide production-ready code if needed.
-Follow clean architecture, SOLID principles, and best practices.

5. Validation

-Explain how to verify the fix.
-Identify potential regressions.
-Suggest additional tests.

6. Optimization

-Recommend improvements related to performance, security, scalability, maintainability, accessibility, and user experience when applicable.

Do not make assumptions. Base conclusions on evidence, and explicitly state when further information is required.
